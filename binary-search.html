Binary Search

Mastering the Divide and Conquer Search Strategy

📌 Deep Dive: Overview

Binary Search is a highly efficient search algorithm that follows the Divide and Conquer paradigm. Instead of scanning through a collection element by element (which is how a Linear Search operates in $O(n)$ time), Binary Search drastically reduces the search space by cutting it in half after every single comparison.

To understand its power, consider searching through an array of one million items. A linear search might take up to one million checks in the worst-case scenario. Binary search, by constantly halving the data, guarantees a result in a maximum of just 20 steps.

⚙️ Prerequisites & Rules

  • Random Access: The underlying data structure must allow for direct index access in $O(1)$ time. This is why binary search is perfect for Arrays, but highly inefficient for Linked Lists.
  • The Golden Rule — Monotonicity: The array must be sorted (either strictly increasing or strictly decreasing). If the array is unsorted, you must sort it first. Keep in mind that the fastest comparison-based sorting algorithms take $O(n \log n)$ time, so sorting an array just to perform a single binary search is counterproductive; it is best used when searching an array multiple times.

🔍 The Algorithm Breakdown

The logic mirrors how you might look up a word in a physical dictionary. You open the book to the middle. If your target word alphabetically precedes the page you are on, you know with absolute certainty it is in the left half of the book. You ignore the right half entirely and repeat the process.

  1. Initialize a low pointer at the 0th index and a high pointer at the final index.
  2. Calculate the middle index: mid = Math.floor((low + high) / 2). (Note: To prevent integer overflow in languages like C++ or Java, use low + (high - low) / 2).
  3. Comparison: Look at the value at the mid index.
    • If array[mid] == target: The search is complete. Return the index.
    • If array[mid] < target: The target must be to the right. Update low = mid + 1.
    • If array[mid] > target: The target must be to the left. Update high = mid - 1.
  4. Repeat steps 2 and 3. The loop terminates when the target is found or when the low pointer crosses the high pointer, meaning the element does not exist in the array.

⏱️ Mathematical Complexity

Time Complexity

  • Best Case: O(1)
    The element we are looking for is exactly at the middle index on the very first calculation.
  • Worst Case: O(log n)
    The target is at the very ends of the array, or doesn't exist, forcing the algorithm to halve the array until only 1 element remains.
  • Average Case: O(log n)

Space Complexity

  • Iterative Implementation: O(1)
    We only need to store three variables in memory (the low, high, and mid pointers) regardless of the size of the array.
  • Recursive Implementation: O(log n)
    While technically doing the same operations, recursion requires memory overhead for the call stack. A new frame is added to the stack for every halving step.
